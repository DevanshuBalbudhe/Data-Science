What is a Time Series?
Introduction & Real-World Applications
📖 Definition

A Time Series is a sequence of data points collected or recorded at successive points in time, usually at uniform intervals. The key idea is that the order of data matters — unlike a normal dataset where rows are independent.

Think of it this way: If I shuffle the rows of a normal Excel sheet of student marks, the dataset is still valid. But if I shuffle a stock price dataset, it becomes completely meaningless — because yesterday affects today.

🧪 Simple Example — Ice Cream Sales

Imagine you run an ice cream shop. You record daily sales:

Real-World Applications

📊 Business & Finance

Stock prices, sales forecasting, inventory management, revenue prediction, customer churn prediction over time.

🌦️ Science & Environment

Weather forecasting, earthquake monitoring, CO₂ levels, tide prediction, climate change analysis.

🏥 Healthcare

Heart rate monitoring (ECG), patient vitals over ICU stay, disease outbreak tracking (COVID case curves).

🌐 Tech & Engineering

Server CPU usage, website traffic, sensor data from IoT devices, energy/power grid demand forecasting.

🔑 Key Characteristic

In time series, past values influence future values. Yesterday's temperature affects today's. Last month's sales shape next month's forecast. This temporal dependency is what makes time series analysis unique and powerful.

Components of Time Series
Trend · Seasonality · Cyclical · Irregular

Any time series can be broken down into 4 fundamental components. Understanding them helps you model, forecast, and explain behavior. Think of it like decomposing a music track into vocals, drums, bass, and effects.

📈
Trend
Long-term increase or decrease in data over a long period
🔄
Seasonality
Regular, predictable pattern that repeats at fixed intervals
〰️
Cyclical
Irregular waves driven by economic or business cycles (years-long)
⚡
Irregular
Random noise — unpredictable shocks and one-off events
Additive Model: Y(t) = Trend + Seasonality + Cyclical + Irregular
Multiplicative:  Y(t) = Trend × Seasonality × Cyclical × Irregular
💡 When to use which?

Use Additive when seasonal variation is roughly constant (e.g., always ±50 units). Use Multiplicative when seasonal variation grows proportionally with the trend (e.g., amplitude increases as data grows — common in sales data).

Identify the Trend

Look at same-quarter values year over year: Q1 went 100→120 (+20), Q2: 130→155 (+25), Q3: 115→140 (+25), Q4: 155→180 (+25). The business is growing ~20–25 Lakhs per year. Trend is upward.

Identify Seasonality

Within each year, the pattern is: Q1 is lowest, Q2 spikes up, Q3 falls slightly, Q4 is highest. This pattern repeats every 4 quarters — that's seasonality (period = 4).

Average Q3 = (115+140)/2 = 127.5 → Index = −9.375
Average Q4 = (155+180)/2 = 167.5 → Index = +30.625

4
Irregular Component

Whatever's left after removing trend and seasonality is random noise (Irregular). For example, if Q2 2022 was expected to be 128 (trend + seasonality) but actual was 130, the irregular part = +2.

Date/Time Handling in Python
pandas datetime · resample · rolling

Python's pandas library is the go-to tool for time series. The three most important concepts are: parsing dates, resampling (changing frequency), and rolling windows (moving averages).

1. datetime — Creating & Parsing Dates

import pandas as pd import numpy as np # Creating a date range (daily frequency) dates = pd.date_range(start='2023-01-01', end='2023-12-31', freq='D') # freq options: 'D'=daily, 'W'=weekly, 'M'=monthly, 'H'=hourly, 'min'=minutely # Creating a time series DataFrame sales = pd.Series( data=np.random.randint(50, 200, size=len(dates)), index=dates ) # Parsing dates from a CSV (very common real use case) df = pd.read_csv('sales.csv', parse_dates=['date'], index_col='date') # Extracting date parts — super useful for feature engineering df['year'] = df.index.year df['month'] = df.index.month df['day_of_week'] = df.index.dayofweek # 0=Monday, 6=Sunday df['quarter'] = df.index.quarter # Slicing by date — very intuitive! jan_data = df['2023-01'] # All of January 2023 q1_data = df['2023-01':'2023-03'] # Q1 2023

2. resample() — Changing Time Frequency

📖 What is Resampling?

Downsampling = Going from fine-grained to coarse (daily → monthly). Upsampling = Going from coarse to fine (monthly → daily, fills with NaN). Think of it like zooming in or out on your data's time axis.

# --- DOWNSAMPLING: Daily → Monthly --- # Sum daily sales into monthly totals monthly_sum = daily_df.resample('M').sum() # Average daily temperature by week weekly_avg = temp_df.resample('W').mean() # Max stock price per quarter quarterly_max = stock_df.resample('Q').max() # Resample frequencies: 'D'=daily, 'W'=weekly, 'M'=month-end # 'MS'=month-start, 'Q'=quarter-end, 'Y'=year-end, 'H'=hourly # --- UPSAMPLING: Monthly → Daily (fill forward) --- daily_from_monthly = monthly_df.resample('D').ffill() # ffill = forward fill (copy last known value forward) # bfill = backward fill, interpolate() = linear interpolation
🧪 Manual Resample Example

Daily sales for Jan Week 1: [Mon:45, Tue:52, Wed:48, Thu:61, Fri:70, Sat:95, Sun:88]

Weekly total = 45+52+48+61+70+95+88 = 459 units

Weekly mean = 459 ÷ 7 = 65.57 units/day average

This is exactly what resample('W').sum() and .mean() would give you!

3. rolling() — Moving Windows

📖 What is a Rolling Window?

A rolling (moving) window computes a statistic over a sliding fixed-size window. It "rolls" through the data. The most common use is Moving Average — smoothing out noise to reveal the underlying trend.

🔑 Why Use Moving Average?

Raw data is messy. A 3-day sale of [40, 80, 40] might just be a random weekend spike. The moving average of 53.3 better represents the true underlying level. Larger windows = smoother = more trend-like. Smaller windows = noisier = more responsive to recent changes.

05
Stationarity
Concept · Why it matters · How to test · How to fix
📖 Definition — Stationarity

A time series is stationary if its statistical properties — mean, variance, and autocorrelation structure — do not change over time. It looks like it fluctuates around a constant level with constant amplitude.

Most machine learning and statistical models (ARIMA, regression, etc.) assume stationarity. If your data is non-stationary (has trends, changing variance), the model will perform poorly. So understanding and achieving stationarity is the foundation of time series modeling.

✅ Stationary Series

Mean = constant (~50) always

Variance = constant throughout

No trend, no seasonality

Example: Daily stock returns (not price!)

❌ Non-Stationary Series

Mean changes over time (upward trend)

Variance may grow or shrink

Has trend and/or seasonality

Example: Company revenue, stock prices

Three Properties of Stationarity

1
Constant Mean

The average value of the series does not increase or decrease over time. If you split the series into halves, both halves should have roughly the same mean.
Non-stationary: Revenue growing from ₹100L to ₹500L → mean is rising.

2
Constant Variance (Homoskedasticity)

The spread of data around the mean stays the same. Non-stationary example: Stock prices that fluctuate by ±5 in 2015 but ±50 in 2023 — variance is increasing. This is called heteroskedasticity.

3
Constant Autocorrelation Structure

The correlation between Y(t) and Y(t-1) is the same regardless of when in the series you are. The "memory" of the series doesn't change over time.

Testing for Stationarity — ADF Test (Manual Intuition)

📊 Augmented Dickey-Fuller (ADF) Test

The ADF test checks the "unit root" hypothesis. A unit root means the series has a trend/random walk (non-stationary). The null hypothesis H₀ = "series has a unit root = non-stationary". We want to reject H₀.

H₀: Series is Non-Stationary (has unit root)
H₁: Series is Stationary

If p-value < 0.05 → Reject H₀ → Series IS Stationary ✅
If p-value > 0.05 → Fail to Reject H₀ → Series is NOT Stationary ❌
from statsmodels.tsa.stattools import adfuller import pandas as pd def adf_test(series, name=''): result = adfuller(series, autolag='AIC') print(f'ADF Test: {name}') print(f' ADF Statistic : {result[0]:.4f}') print(f' p-value : {result[1]:.4f}') print(f' Critical (5%) : {result[4]["5%"]:.4f}') if result[1] < 0.05: print(' ✅ STATIONARY (Reject H0)') else: print(' ❌ NON-STATIONARY (Fail to Reject H0)') # Test original series adf_test(df['sales'], 'Original Sales') # Output might be: # ADF Statistic : -1.23 (not negative enough) # p-value : 0.662 (> 0.05 → non-stationary)

How to Make a Series Stationary — Differencing

Differencing is the most common technique. Instead of using the original values, you compute the change between consecutive values. This removes the trend.

🧪 Manual Differencing Example

Original monthly revenue (₹ Lakhs): 100, 115, 128, 140, 158, 172

1st Difference (Δ = Y(t) − Y(t−1)):

→ 115−100=15, 128−115=13, 140−128=12, 158−140=18, 172−158=14

Differenced values: [15, 13, 12, 18, 14] — these fluctuate around ~14 with no trend! ✅

# Differencing in pandas — matches manual calculation above df['diff_1'] = df['sales'].diff(1) # 1st order differencing df['diff_2'] = df['sales'].diff(2) # 2nd order (if 1st not enough) # Seasonal differencing (remove seasonal pattern) # For monthly data with period=12: subtract value from same month last year df['seasonal_diff'] = df['sales'].diff(12) # Log transformation (if variance is growing with level) import numpy as np df['log_sales'] = np.log(df['sales']) # After transformation, re-test with ADF adf_test(df['diff_1'].dropna(), 'After 1st Differencing') # p-value: 0.001 → ✅ Now stationary!

🔑 Summary — Making Data Stationary

Three main approaches: (1) Differencing — removes trend (most common). (2) Log Transform — stabilizes growing variance. (3) Seasonal Differencing — removes seasonal patterns. In practice, ARIMA models do this automatically — the "I" (Integrated) in ARIMA stands for the order of differencing needed!
