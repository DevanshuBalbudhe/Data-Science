2️⃣ CNN Basics (Convolutional Neural Networks)

🧠 Why CNN for Computer Vision?

Traditional Neural Networks में हर neuron सभी input pixels से connected होता है। एक 224×224 RGB image के लिए = 224×224×3 = 150,528 connections per neuron! यह computationally expensive और overfitting prone है।

CNN की खूबी: Spatial relationships को preserve करते हुए local features extract करता है।

📐 Padding & Stride

A) Padding (Border में Zeros Add करना)

💡 Why Padding?
  • Problem: Convolution से size reduce हो जाती है
  • Solution: Border पर zeros add करो
Manual Example - Padding

Original Image (3×3):

[1 2 3]
[4 5 6]
[7 8 9]

After Padding=1 (5×5):

[0 0 0 0 0]
[0 1 2 3 0]
[0 4 5 6 0]
[0 7 8 9 0]
[0 0 0 0 0]

Types of Padding:

  • Valid: No padding (output size reduces)
  • Same: Padding add karo (output size = input size)

Formula for Same Padding:

Padding = (Filter_Size - 1) / 2
For 3×3 filter: Padding = (3-1)/2 = 1

B) Stride (Filter कितने Steps से Move करे)

🚶 Stride Examples
Stride = 1 (Normal)

Filter एक-एक pixel आगे बढ़ता है

Position 1: [■ ■ ■ □ □]
Position 2: [□ ■ ■ ■ □]
Position 3: [□ □ ■ ■ ■]
Stride = 2 (Skip One)

Filter दो-दो pixels आगे बढ़ता है

Position 1: [■ ■ ■ □ □ □ □]
Position 2: [□ □ ■ ■ ■ □ □]
Position 3: [□ □ □ □ ■ ■ ■]
Output Size Calculation
Output_Size = (Input_Size - Filter_Size + 2×Padding) / Stride + 1

Example 1:

  • Input: 7×7
  • Filter: 3×3
  • Stride: 1
  • Padding: 0
Output = (7 - 3 + 0) / 1 + 1
       = 4 / 1 + 1
       = 5×5

Example 2:

  • Input: 32×32
  • Filter: 5×5
  • Stride: 2
  • Padding: 2
Output = (32 - 5 + 2×2) / 2 + 1
       = (32 - 5 + 4) / 2 + 1
       = 31 / 2 + 1
       = 15.5 + 1
       = 16×16 (floor)

🏊 Pooling (Dimensionality Reduction)

🎯 Pooling क्यों करते हैं?
  • Feature map को छोटा करना (computation reduce)
  • Overfitting control करना
  • Translation invariance (थोड़ी shift से output change नहीं होता)
  • Important features retain रखना

[ 5 6 | 1 3 ]
-----------------
[ 2 8 | 9 1 ]
[ 4 2 | 0 7 ]

Settings:

  • Stride: 2 (non-overlapping)

Step-by-Step Process:

1 Top-Left Region (2×2):
[1 3]
[5 6]

Maximum = 6

2 Top-Right Region:
[2 4]
[1 3]

Maximum = 4

3 Bottom-Left Region:
[2 8]
[4 2]

Maximum = 8

4 Bottom-Right Region:
[9 1]
[0 7]

Maximum = 9

Final Output (2×2):

[6 4]
[8 9]

Size Reduced: 4×4 → 2×2 (75% reduction!)

2) Average Pooling

Same Input (4×4)
1 Top-Left Region:
[1 3]
[5 6] 
Average = (1 + 3 + 5 + 6) / 4
        = 15 / 4
        = 3.75
2 Top-Right Region: 
Average = (2 + 4 + 1 + 3) / 4 = 2.5

Average = (2 + 8 + 4 + 2) / 4 = 4.0

  • Max Pooling: Sharp features retain करता है (edges, textures)
  • Average Pooling: Smooth features retain करता है (backgrounds)
  • Industry में: Max Pooling ज़्यादा popular (90%+ cases)

⚡ Activation Functions

🎯 Activation Functions क्यों?

Without activation functions, neural network सिर्फ linear transformations करेगा। Non-linearity add करने के लिए activation functions ज़रूरी हैं जो complex patterns सीखने में मदद करते हैं।

1) ReLU (Rectified Linear Unit) - Most Popular ⭐

🧮 ReLU Manual Calculation
f(x) = max(0, x) = {
  x, if x > 0
  0, if x ≤ 0
}
Example Values
Input:   [-5,  -2,   0,   3,   7,  -1]
         
Process: max(0,-5) = 0
         max(0,-2) = 0
         max(0, 0) = 0
         max(0, 3) = 3
         max(0, 7) = 7
         max(0,-1) = 0

Output:  [ 0,   0,   0,   3,   7,   0]

Visualization:

    y
    |     /
    |    /
    |   /
____|__/_________ x
    0
    
Slope = 0 for x < 0
Slope = 1 for x > 0

✅ Why ReLU is Popular:

  • Computationally simple (just comparison + max)
  • Solves vanishing gradient problem
  • Sparse activation (many zeros → efficient)
  • Fast training

❌ Problem: "Dying ReLU" - neurons जो हमेशा negative output देते हैं वो मर जाते हैं

2) Sigmoid Function

🧮 Sigmoid Manual Calculation
σ(x) = 1 / (1 + e^(-x))
Step-by-Step Examples
1 x = 0: 
σ(0) = 1 / (1 + e^0)
     = 1 / (1 + 1)
     = 1 / 2
     = 0.5
2 x = 2: 
e^(-2) ≈ 0.135

σ(2) = 1 / (1 + 0.135)
     = 1 / 1.135
     = 0.881
3 x = -2: 
e^2 ≈ 7.389

σ(-2) = 1 / (1 + 7.389)
      = 1 / 8.389
      = 0.119
4 x = 5: 
e^(-5) ≈ 0.0067

σ(5) = 1 / (1.0067)
     = 0.993 ≈ 1

Complete Example:

Input:  [-3,  -1,   0,   1,   3]
Output: [0.05, 0.27, 0.5, 0.73, 0.95]

Properties:

  • Output range: (0, 1) - always positive
  • Smooth gradient (differentiable everywhere)
  • Use case: Binary classification के output layer में

❌ Problem: Vanishing gradient (very large/small inputs पर gradient ≈ 0)

3) Softmax (Multi-class Classification)

🧮 Softmax Complete Manual Calculation
Softmax(x_i) = e^(x_i) / Σ(e^(x_j))
Detailed Example: 3-Class Classification

Input Logits (Raw Scores):

z = [2.0, 1.0, 0.1]

Class 0: z₀ = 2.0
Class 1: z₁ = 1.0
Class 2: z₂ = 0.1
1 Calculate e^x for each class: 
e^(2.0) = 7.389
e^(1.0) = 2.718
e^(0.1) = 1.105
2 Sum of all exponentials: 
Σ = 7.389 + 2.718 + 1.105 = 11.212
3 Divide each by sum: 
P(Class 0) = 7.389 / 11.212 = 0.659 (65.9%)
P(Class 1) = 2.718 / 11.212 = 0.242 (24.2%)
P(Class 2) = 1.105 / 11.212 = 0.099 (9.9%)
4 Verify (sum should be 1): 
0.659 + 0.242 + 0.099 = 1.000 ✅

Final Output:

[0.659, 0.242, 0.099]

Prediction: Class 0 (highest probability) 🎯

3️⃣ R-CNN Basics

🎯 R-CNN क्या है?

R-CNN (Region-based Convolutional Neural Network) ek object detection algorithm है जो image में multiple objects को locate और classify करता है।

Full Form: Region-based Convolutional Neural Network

❓ CNN से R-CNN तक का सफर

❌ CNN की Limitation

CNN सिर्फ बताता है:

  • "Image में क्या है?"
  • Single label: "Dog"

नहीं बताता:

  • "कहाँ है?"
  • "कितने हैं?"
✅ R-CNN का Solution

R-CNN बताता है:

  • "क्या है?" - Class label
  • "कहाँ है?" - Bounding box
  • "कितने हैं?" - Multiple detections

🔑 R-CNN की Core Concepts

1️⃣ Region Proposals

क्या है? Image के वो areas जहाँ object हो सकता है

कैसे generate होते हैं? Selective Search algorithm

कितने? ~2000 regions per image

2️⃣ Feature Extraction

क्या है? हर region से CNN के through features निकालना

कैसे? Region को resize करके CNN में pass करना

Output? Fixed-size feature vector (4096-d)

3️⃣ Classification

क्या है? Feature vector से object का class predict करना

कैसे? SVM (Support Vector Machine) classifier

Output? Class scores (car, person, background)

4️⃣ Bounding Box Regression

क्या है? Initial box को refine/adjust करना

कैसे? Linear regression

Output? Better fitted bounding box

🔬 Selective Search - Region कैसे Separate होते हैं?

🎯 Selective Search Algorithm

यह एक hierarchical grouping algorithm है जो image को over-segment करता है और फिर similar regions को merge करता है।

📸 Selective Search Complete Process
Step-by-Step Region Generation
1 Over-Segmentation (Graph-Based Segmentation) 
Input: 500×500 image

Algorithm: Felzenszwalb's method
Process:
  - Treat each pixel as a node
  - Calculate edge weights (color difference)
  - Merge nodes with similar colors

Output: ~1000-2000 tiny segments

Example visualization:
Original: [Whole image]
          ↓
After:    [S1][S2][S3]...[S1000]
          (tiny segments)

Each segment:
  S1: Pixels (0,0) to (10,8) - Sky region
  S2: Pixels (10,0) to (25,15) - Building
  ...
2 Similarity Calculation 
For every pair of adjacent segments, calculate:

a) Color Similarity:
  - 25-bin color histogram per segment
  - Calculate histogram intersection

Example:
Segment A histogram: [10, 5, 20, 15, ...]
Segment B histogram: [12, 6, 18, 14, ...]

Similarity = Σ min(A[i], B[i]) / Σ A[i]

b) Texture Similarity:
  - SIFT-like features
  - Gaussian derivatives in 8 directions

c) Size Similarity:
  Size(A∪B) - Size(A) - Size(B)
  (prefer merging small regions)

d) Fill Similarity:
  How well A and B fit together
  BBₘᵢₙ(A∪B) - Size(A) - Size(B)

Total Similarity:
S(A,B) = α₁×Color + α₂×Texture + α₃×Size + α₄×Fill
3 Hierarchical Merging 
Initial: 1000 segments

Iteration 1:
  - Find most similar pair (S45, S46)
  - Merge → New segment S1001
  - Regions: 999

Iteration 2:
  - Find next similar pair (S12, S13)
  - Merge → S1002
  - Regions: 998

... Continue until 1 region (whole image)

At each merge:
  - Store bounding box of merged region
  - These become region proposals!

Example progression:
Start:     [S1][S2][S3][S4]...[S1000]
Step 100:  [S1][S1001][S4]...[S800]  (200 regions)
Step 500:  [S2001][S2002]...[S2050]  (50 regions)
Step 999:  [Whole Image]              (1 region)

Total proposals generated: ~2000
4 Diversification Strategy 
To get diverse proposals, run multiple times with:

Different color spaces:
  - RGB
  - HSV
  - LAB
  - Intensity (grayscale)

Different similarity combinations:
  - Color + Texture
  - Color + Size
  - All four

Different segmentation parameters

Final proposals = Union of all runs
Typically: 8 different runs × 250 proposals = ~2000 total
5 Bounding Box Extraction 
For each region:
  - Find minimum bounding rectangle
  - Store coordinates (x, y, width, height)

Example:
Region pixels: {(10,20), (11,20), (10,21), ..., (50,80)}

Bounding box:
  x_min = 10
  y_min = 20
  x_max = 50
  y_max = 80
  
  Box: (10, 20, 40, 60)
       (x,  y,  w,  h)

Quality of Proposals:

Good proposals (IoU > 0.5 with ground truth): ~95%
Excellent proposals (IoU > 0.7): ~70%

Trade-off:
  More proposals → Better coverage but slower
  Fewer proposals → Faster but miss objects

2000 proposals is a sweet spot! ✅

📈 Precision & Recall

🧮 Complete Precision & Recall Calculation
Scenario: Car Detection in 100 Images

Ground Truth:

  • 60 images have cars (Actual Positives)
  • 40 images have no cars (Actual Negatives)

Model Predictions:

  • 50 cars correctly detected (True Positives - TP)
  • 10 cars missed (False Negatives - FN)
  • 5 false car detections (False Positives - FP)
  • 35 correctly identified as no car (True Negatives - TN)

Confusion Matrix:

Predicted: Car Predicted: No Car
Actual: Car TP = 50 FN = 10
Actual: No Car FP = 5 TN = 35
1 Precision Calculation
Precision = TP / (TP + FP) 
Meaning: "जितने detections किए, उनमें से कितने सही थे?"

Precision = 50 / (50 + 5)
          = 50 / 55
          = 0.909 = 90.9%

Interpretation: हर 100 detections में से 91 सही हैं ✅
2 Recall Calculation
Recall = TP / (TP + FN) 
Meaning: "जितने actual cars थे, उनमें से कितने detect हुए?"

Recall = 50 / (50 + 10)
       = 50 / 60
       = 0.833 = 83.3%

Interpretation: 60 में से 50 cars detect हुईं, 10 miss हुईं
3 F1-Score (Harmonic Mean)
F1 = 2 × (Precision × Recall) / (Precision + Recall) 
F1 = 2 × (0.909 × 0.833) / (0.909 + 0.833)
   = 2 × 0.757 / 1.742
   = 1.514 / 1.742
   = 0.869 = 86.9%

Interpretation: Precision और Recall का balanced measure

📝 Precision vs Recall Trade-off:

  • High Precision, Low Recall: Conservative model (misses objects but rarely wrong)
  • Low Precision, High Recall: Aggressive model (detects everything but many false alarms)
  • Ideal: Both high (F1-score close to 1)

🔍 R-CNN Detailed Analysis
Complete Pipeline with Timing
Input: 500×500 image

  Output: 2000 region proposals

  • Very slow (~47 seconds per image)

  • Fixed region proposals (cannot learn them)

Detailed Pipeline
Input: 800×600 image

Step 1: CNN Backbone (Shared) → 0.1 seconds
  VGG16 / ResNet
  Output: Feature Map (50×38×512)

🎯 RPN (Region Proposal Network) - Deep Dive

RPN Architecture & Anchors
1 Anchor Generation 
At each feature map position, place anchors:

Scales: 128², 256², 512² pixels
Aspect Ratios: 1:1, 1:2, 2:1

Total per position: 3 scales × 3 ratios = 9 anchors

Example at position (25, 19) in feature map:
  Corresponds to (400, 304) in original image

Anchor 1: Scale=128, Ratio=1:1
  w = 128, h = 128
  Center: (400, 304)
  Box: (336, 240, 464, 368)

Anchor 2: Scale=128, Ratio=1:2
  w = 128/√2 = 90.5, h = 128×√2 = 181
  Box: (354, 213, 445, 394)

Anchor 3: Scale=256, Ratio=2:1
  w = 256×√2 = 362, h = 256/√2 = 181
  Box: (219, 213, 581, 394)

... 9 anchors total at this position
2 RPN Network Architecture 
Input: Feature Map (50×38×512)
↓
3×3 Convolution (512 filters) → Intermediate layer
↓
Two parallel 1×1 Convolutions:

Branch 1 - Classification:
  Output: 50×38×18 (9 anchors × 2 scores)
  [objectness, background] for each anchor

Branch 2 - Regression:
  Output: 50×38×36 (9 anchors × 4 coordinates)
  [dx, dy, dw, dh] for each anchor

Parameters:
3×3 conv: (3×3×512 + 1) × 512 = 2,359,808
1×1 cls:  (1×1×512 + 1) × 18  = 9,234
1×1 reg:  (1×1×512 + 1) × 36  = 18,468
Total: 2,387,510 parameters
3 Objectness Scoring 
For each anchor, predict 2 scores:

Example anchor #5 at position (25, 19):

Logits: [obj=2.3, bg=-1.2]

Softmax:
  e^2.3 = 9.974
  e^-1.2 = 0.301
  Sum = 10.275

P(object) = 9.974/10.275 = 0.971 (97.1%) ✅
P(background) = 0.301/10.275 = 0.029 (2.9%)

High confidence! Keep this anchor.
4 Bbox Regression in RPN 
Anchor box: (x=336, y=240, w=128, h=128)

RPN predicts offsets:
  dx = +0.05
  dy = -0.03
  dw = +0.1
  dh = +0.08

Transform to actual offsets:
  Δx = dx × w = 0.05 × 128 = 6.4 ≈ 6
  Δy = dy × h = -0.03 × 128 = -3.84 ≈ -4
  Δw = exp(dw) × w = exp(0.1) × 128 = 1.105 × 128 = 141
  Δh = exp(dh) × h = exp(0.08) × 128 = 1.083 × 128 = 139

Refined box:
  x_new = 336 + 6 = 342
  y_new = 240 - 4 = 236
  w_new = 141
  h_new = 139

Refined: (342, 236, 141, 139)
5 NMS (Non-Maximum Suppression) 
17,100 anchors → After objectness filtering: ~2000
2000 proposals → After NMS: ~300

NMS Process:
1. Sort by objectness score (high to low)
2. Take top proposal
3. Remove overlapping (IoU > 0.7)
4. Repeat

Final: ~300 high-quality proposals to detector ✅

✅ Advantages

  • End-to-end training: RPN + Detector together
  • Near real-time: 5-7 FPS on GPU
  • Better proposals: Learned vs hand-crafted
  • GPU acceleration: Everything on GPU
  • High accuracy: mAP ~73% (PASCAL VOC 2007)
  • State-of-the-art in 2015

❌ Limitations

  • Still not truly real-time (need 30+ FPS)
  • Complex training (4-step alternating)
  • Struggles with small objects
  • High GPU memory requirement (12GB+)

🎯 YOLO Detailed Analysis

⚡ YOLO Architecture & Working
YOLO Pipeline (YOLOv3 Example)
1 Grid Division 
Input: 416×416 image

Divide into: 13×13 grid (for large objects)
           + 26×26 grid (for medium objects)
           + 52×52 grid (for small objects)

Example: 13×13 grid
Each cell responsible for detecting objects 
whose center falls in that cell

Grid cell size: 416/13 = 32×32 pixels

Total cells:
13×13 = 169 (large)
26×26 = 676 (medium)
52×52 = 2704 (small)
Total: 3549 cells
2 Each Cell Predicts 
For each cell:
  - 3 bounding boxes (anchors)
  - Each box predicts:
    • (x, y): Center coordinates (relative to cell)
    • (w, h): Width & height (relative to image)
    • Confidence: P(object) × IoU
    • Class probabilities: 80 classes (COCO)

Output per cell:
3 boxes × (5 coords + 80 classes) = 3 × 85 = 255 values

Total output:
13×13×255 + 26×26×255 + 52×52×255
= 43,095 + 172,380 + 689,520
= 904,995 predictions!
3 Manual Prediction Example 
Scenario: Car detected in grid cell (4, 3)

Cell (4, 3) predicts for Box 1:
  tx = 0.6 (sigmoid output)
  ty = 0.4
  tw = 0.3 (before exp)
  th = 0.5
  confidence = 0.92

Convert to actual coordinates:

Cell top-left in image:
  cell_x = 4 × 32 = 128
  cell_y = 3 × 32 = 96

Center coordinates:
  bx = (4 + 0.6) × 32 = 4.6 × 32 = 147.2
  by = (3 + 0.4) × 32 = 3.4 × 32 = 108.8

Assuming anchor: pw = 50, ph = 80

Width & Height:
  bw = pw × e^tw = 50 × e^0.3 = 50 × 1.35 = 67.5
  bh = ph × e^th = 80 × e^0.5 = 80 × 1.65 = 132

Final bounding box:
  Center: (147, 109)
  Size: (68, 132)
  
Convert to (x, y, w, h):
  x = 147 - 68/2 = 113
  y = 109 - 132/2 = 43
  
Box: (113, 43, 68, 132)

Class scores:
  [Car: 0.89, Person: 0.05, Truck: 0.03, ...]
  
Prediction: Car with 92% confidence! ✅
4 Post-Processing (NMS) 
904,995 predictions → Filter by confidence > 0.5
→ ~1000 boxes

Apply NMS:
  IoU threshold: 0.5
  
1000 boxes → After NMS: ~10-50 final detections

Clean output! ✅

🏔️ SSD Architecture - Multi-Scale Detection
SSD300 Pipeline
1 Base Network (VGG16) 
Input: 300×300×3 image
↓
VGG16 convolutions
↓
Multiple feature maps at different scales
2 Multi-Scale Feature Maps 
Layer      Size       Purpose
─────────────────────────────────────────
Conv4_3    38×38      Detect small objects
Conv7      19×19      Detect medium objects
Conv8_2    10×10      Detect large objects
Conv9_2    5×5        Detect very large objects
Conv10_2   3×3        Detect huge objects
Conv11_2   1×1        Detect massive objects

Why multiple scales?
Small object (20×20 px):
  → Best detected in 38×38 map (high resolution)

Large object (200×200 px):
  → Best detected in 5×5 map (low res, but covers it)
3 Default Boxes (Anchors) 
At each location in each feature map:
  - Multiple default boxes
  - Different scales and aspect ratios

Example for 38×38 layer:
  Scales: s_k = 0.2 (60 pixels for 300×300 image)
  Ratios: [1, 2, 3, 1/2, 1/3]
  
  Total per location: 4-6 boxes
  
Total default boxes across all layers:
  38×38×4 + 19×19×6 + 10×10×6 + 5×5×6 + 3×3×4 + 1×1×4
  = 5776 + 2166 + 600 + 150 + 36 + 4
  = 8732 default boxes
4 Predictions Per Box 
For each default box:
  - 4 offset values (dx, dy, dw, dh)
  - (C + 1) class scores (C classes + background)
  
For PASCAL VOC (21 classes including background):
  Each box: 4 + 21 = 25 values
  
Total predictions:
  8732 boxes × 25 = 218,300 values
5 Manual Detection Example 
Scenario: Small person (30×60 pixels)

Which layer detects it?
Conv4_3 (38×38) - high resolution

Position (20, 15) in feature map
→ Corresponds to (158, 119) in original image

Default box #3:
  Scale: 60 pixels
  Ratio: 1:2 (tall)
  Size: 42×84

Network predicts offsets:
  dx = +0.1 → Δx = 0.1 × 42 = 4.2
  dy = -0.05 → Δy = -0.05 × 84 = -4.2
  dw = 0.15 → scale = e^0.15 = 1.16
  dh = 0.08 → scale = e^0.08 = 1.08
  
Refined box:
  Center: (158+4, 119-4) = (162, 115)
  Size: (42×1.16, 84×1.08) = (49, 91)
  
Class scores:
  [Background: 0.05, Person: 0.92, Car: 0.02, ...]
  
Detection: Person at (138, 69, 49, 91) - 92% ✅

🎯 2. Focal Loss (RetinaNet)

🎯 Class Imbalance Problem

Object detection में background examples >>> foreground examples

Example: 10,000 anchors में सिर्फ 50 objects!

📊 Focal Loss Solution
FL(p_t) = -α_t (1 - p_t)^γ log(p_t)
Manual Calculation

Parameters: α = 0.25, γ = 2

1 Easy Example (Background, p=0.9) 
p_t = 0.9 (high confidence, easy to classify)

Standard Cross-Entropy:
  CE = -log(0.9) = 0.105

Focal Loss:
  FL = -0.25 × (1-0.9)^2 × log(0.9)
     = -0.25 × (0.1)^2 × 0.105
     = -0.25 × 0.01 × 0.105
     = -0.000263

Down-weighted! (100x less) ✅
2 Hard Example (Object, p=0.5) 
p_t = 0.5 (low confidence, hard to classify)

Standard Cross-Entropy:
  CE = -log(0.5) = 0.693

Focal Loss:
  FL = -0.25 × (1-0.5)^2 × log(0.5)
     = -0.25 × (0.5)^2 × 0.693
     = -0.25 × 0.25 × 0.693
     = -0.0433

Higher loss! Model focuses here! ✅

Result: Model focuses on hard examples, ignores easy ones!

🎯 3. Feature Pyramid Networks (FPN)

🏔️ FPN Architecture
Multi-Scale Feature Fusion
Problem: Different layers capture different scales

Solution: Combine features from multiple layers

Bottom-up pathway (ResNet):
  C2: 256×256×256  (high resolution, low semantics)
  C3: 128×128×512
  C4: 64×64×1024
  C5: 32×32×2048   (low resolution, high semantics)

Top-down pathway:
  Start from C5, upsample and add to lower levels
  
  P5 = 1×1 conv(C5)          → 32×32×256
  P4 = upsample(P5) + 1×1 conv(C4) → 64×64×256
  P3 = upsample(P4) + 1×1 conv(C3) → 128×128×256
  P2 = upsample(P3) + 1×1 conv(C2) → 256×256×256

Result: All levels have rich semantics + fine details!

Use each level for different scales:
  P2: Small objects (area < 32²)
  P3: Medium objects (32² to 64²)
  P4: Large objects (64² to 128²)
  P5: Very large objects (> 128²)

🎯 5. Hard Negative Mining

Dealing with Class Imbalance
1 Problem 
2000 proposals:
  - 50 positive (IoU > 0.5 with ground truth)
  - 1950 negative (background)

Ratio: 1:39 (severe imbalance!)
2 Solution: Hard Negative Mining 
1. Keep all 50 positives
2. Sort negatives by loss (highest first)
3. Keep only top 150 negatives (high loss = hard)
4. Discard 1800 easy negatives

Final training batch:
  Positives: 50
  Hard negatives: 150
  Ratio: 1:3 (balanced!) ✅

Model learns from mistakes! ✅
