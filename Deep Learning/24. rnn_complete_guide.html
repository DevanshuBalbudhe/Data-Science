🧠 Recurrent Neural Networks (RNN) - Complete Guide

📚 1. Sequential Data aur RNN ki Zaroorat

Sequential Data Kya Hai?

Sequential data wo data hota hai jisme order matter karta hai. Har data point previous data points par depend karta hai.

Examples of Sequential Data:

  • Text: "मैं खाना खा रहा हूं" - har shabd previous shabd par depend karta hai
  • Time Series: Stock prices, weather data - current value past values par depend karta hai
  • Speech: Audio signals - har frame pichle frames se connected hai
  • Video: Consecutive frames ek sequence banate hain

ANN/CNN ki Limitations Sequential Data Mein

❌ ANN Problems:

  • Fixed input size chahiye
  • Temporal dependencies nahi samajh sakta
  • Order of inputs matter nahi karta
  • No memory of previous inputs

❌ CNN Problems:

  • Spatial patterns ke liye best, temporal nahi
  • Long-range dependencies capture nahi kar sakta
  • Sequential order important nahi hota
  • Fixed receptive field

Temporal Dependencies Kya Hain?

Temporal dependencies matlab current output previous inputs par depend karta hai. Example:

Sentence: "The cat sat on the ___"

Yahan blank word fill karne ke liye humein puri sentence ka context chahiye. "mat" ya "chair" sensible hai kyunki "cat" aur "sat on" pehle aaya hai.

🏗️ 2. RNN Architecture aur Working

Basic RNN Structure

RNN ek special type ka neural network hai jo loops use karta hai. Ye information ko persist kar sakta hai.

RNN Unfolded Structure:

x1
→
h1
→
y1


x2
→
h2
→
y2


x3
→
h3
→
y3

x = Input, h = Hidden State, y = Output

⬆️ Hidden states ek dusre se connected hain (temporal connection)

Hidden State Concept

Hidden state RNN ki "memory" hai. Ye previous information ko store karta hai aur current input ke saath combine hota hai.

Hidden State Formula:

ht = tanh(Whh × ht-1 + Wxh × xt + bh)
Output Formula:

yt = Why × ht + by

Components Explanation:

  • ht: Current time step ka hidden state
  • ht-1: Previous time step ka hidden state (memory)
  • xt: Current input
  • Whh: Hidden-to-hidden weight matrix
  • Wxh: Input-to-hidden weight matrix
  • Why: Hidden-to-output weight matrix
  • bh, by: Bias terms
  • tanh: Activation function (-1 se 1 ke beech output deta hai)

📊 Manual Calculation Example

Problem: Simple RNN for Sequence "HELLO"

Maan lo humein "HELLO" word process karna hai. Har letter ek time step hai.

Step 1: Input Encoding

H = [1, 0, 0, 0, 0]

E = [0, 1, 0, 0, 0]

L = [0, 0, 1, 0, 0]

O = [0, 0, 0, 0, 1]

Step 2: Initialize Parameters

Input dimension = 5, Hidden dimension = 3


Wxh (5×3):

[0.1 0.2 -0.1]
[0.3 -0.2 0.4]
[-0.1 0.1 0.2]
[0.2 0.3 -0.2]
[0.1 -0.1 0.3]

Whh (3×3):

[0.5 -0.3 0.2]
[0.1 0.4 -0.1]
[-0.2 0.2 0.6]

bh = [0.1, 0.1, 0.1]

h0 = [0, 0, 0] (initial hidden state)

Step 3: Time Step t=1 (Input: H)

x1 = [1, 0, 0, 0, 0]


Wxh × x1 = [0.1, 0.3, -0.1]

Whh × h0 = [0, 0, 0]

Sum = [0.1, 0.3, -0.1] + [0.1, 0.1, 0.1] = [0.2, 0.4, 0.0]


h1 = tanh([0.2, 0.4, 0.0]) = [0.197, 0.380, 0.0]

Step 4: Time Step t=2 (Input: E)

x2 = [0, 1, 0, 0, 0]


Wxh × x2 = [0.2, -0.2, 0.1]

Whh × h1 = [0.5×0.197 + (-0.3)×0.380 + 0.2×0, ...] = [-0.016, 0.171, 0.037]

Sum + bias = [0.2, -0.2, 0.1] + [-0.016, 0.171, 0.037] + [0.1, 0.1, 0.1]

= [0.284, 0.071, 0.237]


h2 = tanh([0.284, 0.071, 0.237]) = [0.277, 0.071, 0.233]

Key Point: Dekho kaise h2 calculate karte waqt h1 use hua. Ye hi RNN ki memory hai!

🔄 3. Forward Propagation in RNN

Complete Forward Pass Algorithm

Step 1

Initialize: h0 = 0 (ya random values)

Step 2

For each time step t:

  • Input xt lo
  • Previous hidden state ht-1 lo
  • Calculate: ht = tanh(Whhht-1 + Wxhxt + bh)
  • Calculate: yt = Whyht + by
Step 3

Loss Calculate karo: L = Σ Loss(yt, targett)

📝 Detailed Example: Sentiment Analysis

Task: "Movie bahut acchi thi" → Positive (1) ya Negative (0)?

Vocabulary: {Movie:0, bahut:1, acchi:2, thi:3, buri:4}

Word Embeddings (2D):

Movie → [0.5, 0.2]

bahut → [0.3, 0.8]

acchi → [0.9, 0.7]

thi → [0.1, 0.3]

Parameters:

Hidden size = 2

Wxh = [[0.4, 0.3], [0.2, 0.5]]

Whh = [[0.6, -0.2], [0.1, 0.7]]

Why = [0.8, 0.5]

bh = [0, 0], by = 0

t=1: "Movie"

x1 = [0.5, 0.2]

h0 = [0, 0]


Linear: 0.4×0.5 + 0.2×0.2 = 0.24 (first neuron)

Linear: 0.3×0.5 + 0.5×0.2 = 0.25 (second neuron)

h1 = tanh([0.24, 0.25]) ≈ [0.236, 0.245]

t=2: "bahut"

x2 = [0.3, 0.8]

h1 = [0.236, 0.245]


From input: 0.4×0.3 + 0.2×0.8 = 0.28

From hidden: 0.6×0.236 + 0.1×0.245 = 0.166

Total first neuron: 0.28 + 0.166 = 0.446


From input: 0.3×0.3 + 0.5×0.8 = 0.49

From hidden: -0.2×0.236 + 0.7×0.245 = 0.125

Total second neuron: 0.49 + 0.125 = 0.615


h2 = tanh([0.446, 0.615]) ≈ [0.418, 0.548]

Final Output (after "thi"):

Suppose h4 = [0.6, 0.7]

y = Why × h4 = 0.8×0.6 + 0.5×0.7 = 0.83

Probability = sigmoid(0.83) = 0.696 ≈ Positive!

⏮️ 4. Backpropagation Through Time (BPTT)

BPTT Kya Hai?

BPTT normal backpropagation ka extended version hai jo time ke through unfold hota hai. Ye gradients ko past time steps tak propagate karta hai.

BPTT Gradient Flow:

Loss ← y3 ← h3 ← h2 ← h1 ← h0

Gradients backward propagate hote hain through all time steps

BPTT Mathematical Formulation

∂L/∂Whh = Σt ∂Lt/∂Whh

Har time step par gradient calculate karna padta hai aur unhe sum karna padta hai.

📐 BPTT Manual Calculation

Simple 3-Step Sequence

Given:

Sequence length T = 3

Predictions: ŷ1=0.3, ŷ2=0.6, ŷ3=0.8

Actual: y1=0, y2=1, y3=1

Loss function: MSE = (ŷ - y)²

Step 1: Calculate Loss

L1 = (0.3 - 0)² = 0.09

L2 = (0.6 - 1)² = 0.16

L3 = (0.8 - 1)² = 0.04

Total Loss L = 0.09 + 0.16 + 0.04 = 0.29

Step 2: Gradient at t=3

∂L3/∂ŷ3 = 2(ŷ3 - y3) = 2(0.8 - 1) = -0.4

∂ŷ3/∂h3 = Why (suppose = 0.5)

∂L3/∂h3 = -0.4 × 0.5 = -0.2

Step 3: Gradient at t=2 (Chain Rule)

h3 depends on h2, so:

∂L/∂h2 = ∂L2/∂h2 + ∂L3/∂h3 × ∂h3/∂h2


∂h3/∂h2 = Whh × (1 - tanh²(h3))

Suppose Whh = 0.6, tanh²(h3) ≈ 0.64

= 0.6 × (1 - 0.64) = 0.216


∂L2/∂h2 = 2(0.6-1) × 0.5 = -0.4

∂L/∂h2 = -0.4 + (-0.2) × 0.216 = -0.443

Step 4: Weight Update

Learning rate α = 0.01

∂L/∂Whh ≈ -0.5 (calculated from all gradients)


Whh(new) = Whh(old) - α × ∂L/∂Whh

Whh(new) = 0.6 - 0.01 × (-0.5) = 0.605

Dhyan do: Gradients ko har time step ke through propagate karna padta hai. Ye computational cost increase karta hai!

⚠️ 5. Key Problems: Vanishing & Exploding Gradients

Vanishing Gradient Problem

Jab gradients backward propagate hote hain through time, wo repeatedly multiply hote hain. Agar values < 1 hain, to gradients exponentially chhote ho jaate hain.

∂ht/∂h0 = ∏i=1 to t Whh × tanh'(...)

Example:

Suppose Whh = 0.5, aur tanh' ≈ 0.4

Gradient multiplication per step = 0.5 × 0.4 = 0.2


After 5 steps: 0.2⁵ = 0.00032 (bahut chhota!)

After 10 steps: 0.2¹⁰ = 0.0000001024 (almost zero!)


Problem: Early layers kuch seekh nahi pate kyunki unka gradient nearly zero ho jata hai.

Exploding Gradient Problem

Opposite case: agar weights > 1 hain, to gradients exponentially badh jaate hain.

Example:

Suppose Whh = 2.0, aur tanh' ≈ 0.9

Gradient multiplication per step = 2.0 × 0.9 = 1.8


After 5 steps: 1.8⁵ = 18.89

After 10 steps: 1.8¹⁰ = 357.05

After 20 steps: 1.8²⁰ = 127,514!


Problem: Weights bahut bade ho jaate hain, training unstable ho jata hai, NaN values aa sakte hain.

Solutions

Problem Solution How it Works
Vanishing Gradient LSTM/GRU Gates use karke gradient flow control karte hain
Exploding Gradient Gradient Clipping Gradients ko max value par cap kar dete hain
Both Better Initialization Xavier/He initialization use karte hain
Both Residual Connections Skip connections add karte hain

Gradient Clipping Example:

Threshold = 5.0

Calculated gradient = 12.5


if ||gradient|| > threshold:

    gradient = gradient × (threshold / ||gradient||)


Clipped gradient = 12.5 × (5.0 / 12.5) = 5.0

🚀 6. Advanced RNN Variants

A. LSTM (Long Short-Term Memory)

LSTM sabse popular RNN variant hai. Ye vanishing gradient problem solve karta hai using gates.

LSTM Cell Structure:

3 Gates:

  1. Forget Gate (ft): Purani memory ko kitna bhoolna hai
  2. Input Gate (it): Nayi information ko kitna add karna hai
  3. Output Gate (ot): Cell state se kitna output dena hai

Cell State (Ct): Long-term memory highway

LSTM Equations:

Forget Gate:

ft = σ(Wf · [ht-1, xt] + bf)


Input Gate:

it = σ(Wi · [ht-1, xt] + bi)

C̃t = tanh(WC · [ht-1, xt] + bC)


Cell State Update:

Ct = ft ⊙ Ct-1 + it ⊙ C̃t


Output Gate:

ot = σ(Wo · [ht-1, xt] + bo)

ht = ot ⊙ tanh(Ct)


σ = sigmoid function, ⊙ = element-wise multiplication

📊 LSTM Manual Calculation

Simple LSTM Forward Pass (1 time step)

Given:

xt = [0.5]

ht-1 = [0.2]

Ct-1 = [0.8]

All weights = 1.0, All biases = 0 (simplified)

Step 1: Forget Gate

Concatenated input = [ht-1, xt] = [0.2, 0.5]

ft = σ(1.0 × 0.2 + 1.0 × 0.5) = σ(0.7)

ft = 1/(1 + e-0.7) = 0.668

Meaning: 66.8% purani memory retain karenge

Step 2: Input Gate

it = σ(1.0 × 0.2 + 1.0 × 0.5) = σ(0.7) = 0.668

C̃t = tanh(1.0 × 0.2 + 1.0 × 0.5) = tanh(0.7) = 0.604

Meaning: Nayi candidate value = 0.604

Step 3: Cell State Update

Ct = ft ⊙ Ct-1 + it ⊙ C̃t

Ct = 0.668 × 0.8 + 0.668 × 0.604

Ct = 0.534 + 0.403 = 0.937

Purani memory ka kuch part + nayi information

Step 4: Output Gate

ot = σ(1.0 × 0.2 + 1.0 × 0.5) = 0.668

ht = ot ⊙ tanh(Ct)

ht = 0.668 × tanh(0.937)

ht = 0.668 × 0.733 = 0.490

Final hidden state output!

B. GRU (Gated Recurrent Unit)

GRU LSTM ka simplified version hai. Sirf 2 gates hain instead of 3. Faster training, kam parameters.

Reset Gate:

rt = σ(Wr · [ht-1, xt])


Update Gate:

zt = σ(Wz · [ht-1, xt])


Candidate Hidden State:

h̃t = tanh(W · [rt ⊙ ht-1, xt])


Final Hidden State:

ht = (1 - zt) ⊙ ht-1 + zt ⊙ h̃t

LSTM vs GRU Comparison:

Feature LSTM GRU
Gates 3 (Forget, Input, Output) 2 (Reset, Update)
Cell State Separate cell state + hidden state Only hidden state
Parameters More (slower) Fewer (faster)
Performance Better for complex sequences Better for simpler tasks
Training Time Longer Shorter

C. Bidirectional RNN

Do RNN use karte hain - ek forward, ek backward. Dono directions ka context milta hai.

Bidirectional Structure:

Forward: x1 → x2 → x3 → hfwd

Backward: x3 ← x2 ← x1 → hbwd

Final: h = [hfwd, hbwd]

Use case: Jab sentence completion mein dono taraf ka context chahiye. Example: "The _____ is blue" - pata nahi "sky" hai ya "ocean" jab tak pure sentence ko nahi dekhte.

D. Deep RNN (Stacked RNN)

Multiple RNN layers ek ke upar ek. Hierarchical features learn karte hain.

Layer 3: h3,1 → h3,2 → h3,3

↑          ↑          ↑

Layer 2: h2,1 → h2,2 → h2,3

↑          ↑          ↑

Layer 1: h1,1 → h1,2 → h1,3

↑          ↑          ↑

Input: x1     x2     x3

💡 7. Practical Applications of RNN

1. Sentiment Analysis

Problem: Review positive hai ya negative?

Input: "This movie was amazing and thrilling!"

Output: Positive (1)


Architecture:

  • Words ko embeddings mein convert karo
  • LSTM se sequence process karo
  • Last hidden state se classification karo
  • Sigmoid output: 0-1 (negative-positive)

2. Language Modeling

Task: Next word predict karna

Input: "I am going to the"

Output: "market" (probability distribution)


How: RNN har position par vocab ke har word ka probability deta hai.

3. Machine Translation (Seq2Seq)

English to Hindi:

Input: "I love you"

Output: "मैं तुमसे प्यार करता हूं"


Architecture:

  1. Encoder RNN: Input sentence ko context vector mein encode karo
  2. Decoder RNN: Context vector se output sequence generate karo
  3. Attention: Har output word ke liye relevant input words par focus karo

4. Text Generation

Character-level RNN:

Training: Shakespeare ki poetry

Generated: "To be or not to be, that is the question..."


Process:

  • Start token 'T' se shuru karo
  • RNN next character 'o' predict kare
  • 'To' par next ' ' (space) predict kare
  • Continue until end token

5. Time Series Forecasting

Stock Price Prediction:

Input: Last 30 days prices [100, 102, 98, 105, ...]

Output: Tomorrow's price → 107


Features: Price, volume, moving averages

Model: LSTM with multiple features input

6. Speech Recognition

Audio → Text:

Input: Audio waveform features (MFCC)

Output: "Hello how are you"


Architecture: Bidirectional LSTM + CTC Loss

7. Video Activity Recognition

Task: Video mein kya ho raha hai?

Input: Video frames sequence

Output: "Playing football", "Dancing", etc.


Method:

  • CNN se har frame ka feature extract karo
  • RNN se temporal sequence samjho
  • Final classification karo

8. Named Entity Recognition (NER)

Sentence: "Sachin Tendulkar plays for India in Mumbai."

Output:

  • Sachin Tendulkar → PERSON
  • India → COUNTRY
  • Mumbai → CITY

Architecture: Bidirectional LSTM + CRF layer

✅❌ 8. RNN Pros and Cons

✅ Advantages (Pros)

  • Sequential Processing: Time-based data ko naturally handle kar sakta hai
  • Variable Length Input: Koi bhi length ka sequence process kar sakta hai
  • Memory: Previous information ko store karke use kar sakta hai
  • Parameter Sharing: Same weights har time step par, so kam parameters
  • Context Understanding: Pura sequence ka context samajh sakta hai
  • Versatile: Multiple tasks ke liye use ho sakta hai (NLP, time series, etc.)
  • Temporal Dependencies: Long-term patterns capture kar sakta hai (LSTM/GRU)

❌ Disadvantages (Cons)

  • Vanishing Gradient: Long sequences mein gradient disappear ho jata hai
  • Exploding Gradient: Weights bahut bade ho sakte hain
  • Sequential Processing: Parallel processing nahi kar sakte, slow training
  • Long-term Memory: Basic RNN bahut long dependencies nahi sambhal sakta
  • Computationally Expensive: LSTM/GRU bahut resources lete hain
  • Difficult to Train: Hyperparameter tuning challenging hai
  • Limited by Sequential Nature: Transformers jaise models se slow

When to Use RNN vs Other Models?

Use Case Best Model Reason
Short sequences (<100 words) RNN/LSTM Simple, fast, sufficient
Long sequences (>500 words) Transformer Better long-range dependencies
Time series forecasting LSTM/GRU Temporal patterns capture
Real-time processing GRU Faster than LSTM
Translation/Summarization Transformer Parallel processing, attention
Speech recognition Bi-LSTM Both direction context

🔧 9. Implementation Tips

Common Hyperparameters:

Parameter Typical Range Effect
Hidden Size 128 - 512 Model capacity, memory
Number of Layers 1 - 4 Model depth, complexity
Learning Rate 0.001 - 0.01 Training speed, convergence
Dropout 0.2 - 0.5 Regularization, overfitting
Batch Size 32 - 128 Training stability, speed
Gradient Clip 5.0 - 10.0 Prevent exploding gradients

Best Practices:

  • Gradient Clipping: Always use for RNN/LSTM
  • Batch Normalization: Helps with training stability
  • Dropout: Between layers for regularization
  • Xavier/He Initialization: Better than random
  • Teacher Forcing: Use during training for seq2seq
  • Attention Mechanism: Improves long sequences
  • Bidirectional: When full sequence available
